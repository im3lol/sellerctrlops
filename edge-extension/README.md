# SellerCtrl — السحب الذكي (Edge Extension)

إضافة لمتصفح **Microsoft Edge** (MV3) لالتقاط بيانات المنتجات تفاعلياً من موقع
العميل: تفتح صفحة المنتج، تضغط على العنصر (الاسم/الصورة/السعر…)، فتلتقط الإضافة
الـ CSS selector وتعرض النتيجة في لوحة جانبية بهوية SellerCtrl. ثم «حفظ وتشغيل»
ينشئ مهمة سحب يشغّلها عامل Docker على كل المنتجات المسودة ويحفظ البيانات الناقصة
في المنصة.

## التثبيت على Edge

1. افتح `edge://extensions`.
2. فعّل **وضع المطوّر (Developer mode)** من الأسفل يسار.
3. اضغط **تحميل غير مضغوط (Load unpacked)** واختر مجلد `edge-extension/`.
4. ثبّت الإضافة على شريط الأدوات (اختياري).

## الإعداد

1. اضغط أيقونة الإضافة ← من اللوحة اختر **فتح الإعدادات** (أو زر اليمين ← خيارات).
2. أدخل:
   - **رابط المنصة**: `https://sellerctrlops.vercel.app`
   - **رمز السحب**: نفس قيمة `SCRAPER_TOKEN` المضبوطة في Vercel.
   - **معرّف مساحة العمل**: انسخه من رابط مساحة العمل `…/workspaces/<UUID>`.
3. اضغط **اختبار الاتصال** ثم **حفظ**.

## الاستخدام

1. ارفع منتجات العميل عبر Excel في **وضع المسودة** (لينك المنتج فقط مثلاً).
2. افتح اللوحة الجانبية للإضافة ← **تحميل** المنتجات المسودة.
3. **فتح أول منتج وبدء الالتقاط** → تُفتح صفحة المنتج.
4. لكل حقل اضغط **حدّد** ثم اضغط على العنصر المقابل في الصفحة (تظهر النتيجة).
5. **حفظ وتشغيل** → عامل Docker يفتح كل لينك ويحفظ البيانات الناقصة أول بأول.
6. ارجع للمنصة، راجع المنتجات المسودة، واضغط **تأكيد وإتاحة للموظفين**.

> العامل (Docker) لازم يكون شغّالاً — راجع `scraper-worker/README.md`.
> البيانات المسحوبة تملأ الحقول الفارغة فقط ولا تستبدل بيانات موجودة، ويظل المنتج
> مسودة حتى يؤكّده مراجع.
